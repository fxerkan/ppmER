🗂️ PPM Data Assistant

AI-powered natural language interface for your Jira Project & Portfolio Management data

What is PPM Data Assistant?

PPM Data Assistant lets you explore your Jira project data by simply asking questions in plain English (or Turkish). The assistant automatically writes SQL, queries the data warehouse, generates charts, and surfaces key insights — no SQL knowledge required.

What Can It Do?

✅ Natural language queries — ask in any language, get data back
✅ Auto-visualization — bar, pie, and line charts generated automatically
✅ Key Insights — 2-3 AI-generated findings after every data result
✅ Metabase integration — browse and run saved Metabase reports/questions
✅ Multi-format download — export results as CSV, Excel, or Markdown
✅ Conversation history — all chats saved, resumable from left panel
✅ Multiple AI models — switch between Gemini, OpenAI, DeepSeek, and more
❌ Cannot modify data (no inserts, updates, or deletes)
❌ Cannot access real-time Jira data (data is synced via scheduled pipelines)
❌ Cannot predict future values or forecast
❌ Cannot access data outside the configured warehouse schemas

Output Formats

1. Table + Insights (most common)

How many open issues are there per project?
Returns a markdown table:

| project_name | open_issues |
|---|---|
| Alpha | 42 |
| Beta | 18 |

Followed by a 💡 Key Insights section with AI-generated findings.
Output types: 📋 Table 💡 Insights 📥 Downloads

2. Bar Chart

Show me the top 10 users by logged hours this month
Returns a table + interactive bar chart showing user names vs hours. Good for comparing categorical values.
Output types: 📋 Table 📊 Bar Chart 💡 Insights

3. Pie Chart

What's the distribution of issues by status?
When there are ≤7 categories and 1 numeric column, a donut/pie chart is generated automatically.
Output types: 📋 Table 🥧 Pie Chart 💡 Insights

4. Trend / Line Chart

How have logged hours changed month by month over the last year?
When results contain a date/time column, a line chart is generated to show the trend over time.
Output types: 📋 Table 📈 Line Chart 💡 Insights

5. Metabase Report Results

What Metabase reports do we have about project health?
The assistant calls list_metabase_questions and returns a list of saved questions with their IDs.

You can then say: "Run report #15" or "Show me the data from the project health card"
Run the missing effort report from Metabase
The assistant finds the matching Metabase question, calls run_metabase_question, and returns the data — already pre-validated by your BI team.
Output types: 📋 Table 📊 Chart (if applicable) 💡 Insights

6. Downloads

After any response containing data, you'll see inline download links below the message:

📥 Download: CSV · Excel · Markdown

Click to download the raw result in your preferred format.

How to Use Metabase Integration

  1. Discover reports: Ask "List all Metabase questions" or "What reports do we have about budgets?"
  2. Run a specific report: Say "Run Metabase question 42" or "Show me the data from the portfolio health report"
  3. Browse dashboards: Ask "What Metabase dashboards are available?"
  4. Compare with SQL: You can ask the assistant to cross-reference a Metabase result with a custom SQL query
💡 Tip: Metabase questions are pre-built and validated by your BI team — use them when you need trusted, standardized metrics. Use direct SQL queries for ad-hoc exploration.

Tips for Best Results

🎯 Be specific: Instead of "show hours", ask "show logged hours per project for October 2025"

🔄 Follow up: After a result, ask "Now filter this to only projects with >100 hours"

📊 For charts: Ask questions that produce 2-column results (category + number) for best visualizations

🏛️ For Metabase: First run "list metabase questions" to discover what's available

🌍 Any language: Ask in Turkish, English, or any language — the assistant responds in the same language
